**Companion Project:** [Customer Win-Back Engine](https://github.com/JustineDataEng/Retail-Intelligence-System)

---

## Project Overview

This project builds a full SQL analytics layer on top of the Online Retail II dataset, the same dataset used in the Customer Win-Back Engine. While that project handled data ingestion, cleaning, and RFM segmentation in Python, this project demonstrates the same logic implemented entirely in PostgreSQL using production-grade SQL patterns.

The goal is to show how the same business intelligence work (customer segmentation, churn risk, revenue analysis, win-back targeting) can be expressed as scalable, reusable SQL that lives in a database rather than a Python notebook.

## SQL Concepts Demonstrated

**Window Functions**
- `NTILE(5)` for RFM scoring across recency, frequency, and monetary dimensions
- `RANK()`, `DENSE_RANK()`, `ROW_NUMBER()` for customer and product rankings
- `LAG()` and `LEAD()` for month-over-month revenue growth calculations
- `FIRST_VALUE()` and `LAST_VALUE()` for first and last purchase tracking
- Running totals and 3-month moving averages using `ROWS BETWEEN`
- `NTILE(100)` for customer lifetime value percentile ranking

**CTEs (Common Table Expressions)**
- Multi-step RFM pipelines broken into readable, reusable stages
- Cohort retention analysis using layered CTEs
- Win-back priority scoring combining LTV decile and recency decile

**Aggregations and Filtering**
- Revenue concentration analysis (top customers, top products, revenue by country)
- Churn risk tiering using `CASE` with inactive day thresholds
- Revenue recovery modelling at 10%, 20%, and 30% win-back conversion rates
- Personalised product recommendations per at-risk customer using `RANK()` within partition

## Key Findings (from the dataset)

- The dataset covers transactions from December 2009 to December 2011 across 40+ countries
- The United Kingdom accounts for the majority of revenue, with Germany, France, and EIRE as secondary markets
- A small percentage of customers (Champions) typically drive a disproportionate share of total revenue
- Cohort retention analysis reveals typical e-commerce drop-off patterns, with Month 0 (first purchase) showing the highest activity and sharp declines in Month 1 and 2
- Win-back revenue modelling shows significant recovery potential even at conservative 10% re-engagement rates
